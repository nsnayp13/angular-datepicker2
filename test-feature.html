Test showPrevNextDaysInOneMonth Feature

Feature Requirements Verification

  • ✅ Added showPrevNextDaysInOneMonth input property to main calendar component
  • ✅ Updated Day interface to include isPrevNextMonth property for styling
  • ✅ Modified month service to generate prev/next month days when in single month mode
  • ✅ Updated day view template to show prev/next month days with opacity 0.85
  • ✅ Added CSS styles for prev/next month days with opacity 0.85
  • ✅ Added showPrevNextDaysInOneMonth setting to GitHub pages example

How to Test

  1. Open the development server at http://localhost:4200
  2. Set viewMode to "1" (single month mode)
  3. Find the "showPrevNextDaysInOneMonth" setting in the props table
  4. Toggle it to "Show prev/next month days"
  5. Observe that the calendar now fills empty cells with previous/next month days
  6. Verify that these days have reduced opacity (0.85)
  7. Switch to multi-month mode (viewMode > 1) and verify the feature doesn't apply

Expected Behavior

  • Single Month Mode (viewMode = 1) with feature enabled: Calendar shows prev/next month days to fill the grid, with opacity 0.85
  • Single Month Mode (viewMode = 1) with feature disabled: Calendar shows only current month days (original behavior)
  • Multi-Month Mode (viewMode > 1): Feature has no effect, shows original behavior regardless of setting
