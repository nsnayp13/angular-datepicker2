Тест обновления календаря при изменении weekStart и weekends

Проверенные изменения:

✅ ngOnChanges обновлен: Добавлена обработка изменений weekStart и weekends
✅ CalendarService улучшен: Добавлены реактивные методы setWeekStart() и setWeekends()
✅ Реактивные обновления: Добавлены BehaviorSubject для weekStart и weekends
✅ MonthViewComponent обновлен: Подписка на изменения weekStart для автоматического пересчета недель
✅ DayService улучшен: Подписка на изменения weekends для обновления статуса выходных дней

Исправленные проблемы:

✅ Проблема: При изменении weekStart календарь не обновлялся
Решение: Добавлен метод _weekStart() в ngOnChanges, который вызывает setWeekStart() в сервисе
✅ Проблема: При изменении weekends календарь не обновлялся
Решение: Добавлен метод _weekends() в ngOnChanges, который вызывает setWeekends() в сервисе
✅ Проблема: Неэффективное обновление календаря
Решение: Использование реактивных подходов с BehaviorSubject и автоматическое обновление только при реальных изменениях

Как тестировать:

  1. Запустите приложение: npx ng serve
  2. Откройте браузер на http://localhost:4200
  3. Измените "weekStart" в выпадающем списке - календарь должен обновиться
  4. Измените "weekends" в выпадающем списке - календарь должен обновиться
  5. Проверьте, что выходные дни правильно выделяются после изменения
  6. Проверьте, что первый день недели изменяется при изменении weekStart

Технические улучшения:

  • Реактивность: Использование BehaviorSubject для отслеживания изменений
  • Эффективность: Обновление только при реальных изменениях значений
  • Автоматизация: Автоматический пересчет календаря при изменении параметров
  • Современные подходы: Замена устаревших методов на современные реактивные паттерны
